# 2026-03-30

- Investigated `5.pdf` (E.SUN statement). Using local `pdfjs-dist` in Node, both pages returned `itemCount = 0`, confirming this file is image-only and not a text-layer PDF.
- The observed failure was therefore OCR-selection-related, not a simple E.SUN regex miss on extracted text.
- Added E.SUN-specific OCR support in `apps/web/src/lib/pdf-parser.ts`:
  - more OCR candidates (`raw`, `enhanced`, `thresholded`)
  - E.SUN-focused crop regions that bias toward lower transaction-table areas instead of the cover summary
  - E.SUN OCR scoring that rewards transaction-like rows/headers and penalizes cover-page markers such as `綜合對帳單`, `資料參考日`, and客服資訊
- Fixed a routing bug in Gmail PDF import: bank statements were previously always sent through the Sinopac bank-statement parser. E.SUN statements now use an explicit E.SUN bank-statement path, and E.SUN OCR also uses per-page cover-page skipping similar to the CTBC flow.
- Fixed E.SUN bank-statement row parsing for ROC-date lines such as `115/02/05 ...`; the shared bank-statement parser now accepts `YYY/MM/DD` in addition to `YYYY/MM/DD`, which was blocking OCR-recovered E.SUN transaction rows from being imported.
- Added `parseEsunLoanSection(...)` so E.SUN statement imports can persist loan snapshot rows from the dedicated loan section (`資料日期`, masked loan account, remaining balance) instead of only importing account transactions.
- Broadened the E.SUN loan masked-account matcher to accept 6-8 leading digits before `***`, after finding a real sample like `0484165***406` that was still being skipped.
- Adjusted E.SUN OCR page assembly so the importer can retain multiple useful OCR regions from the same page instead of only the single highest-scoring crop; this is intended to preserve lower loan/insurance summary sections when upper transaction rows otherwise dominate the score.
- Relaxed `parseEsunLoanSection(...)` again to handle OCR-fragmented loan headers (`貸款 資料日期...` on one line) and spaced masked loan accounts like `0 4 8 4 1 6 5 * * * 4 0 6`.
- Added a full-text fallback in `parseEsunLoanSection(...)` so E.SUN loan rows can still be extracted even when OCR does not preserve a clean standalone `貸款` section block.
