# 2026-03-29

- Fixed a Sinopac insurance parser regression where statement imports could miss insurance snapshots even though the bank statement itself imported successfully.
- Root cause: `parseSinopacInsuranceSection` was too strict about section and policy boundaries, and could miss records when `保險`, `非投資型`, and the policy number were collapsed onto the same line.
- Relaxed the parser to detect nearby insurance context, support inline policy/type rows, and avoid treating the insured person's masked ID as a new policy.
- Added a regression test for inline Sinopac insurance rows in `apps/api/tests/pdf-credit-card.test.ts`.
- Verification: `npm --workspace @hearth/api run build`, `npm --workspace @hearth/web run check`, plus direct Node invocation of `parseSinopacInsuranceSection(...)` with a Unicode-escaped inline Sinopac sample (test runner in this sandbox still hits `spawn EPERM`).
