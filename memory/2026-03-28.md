# 2026-03-28

- Fixed loan and insurance detail deletion/render drift in the web UI.
- Root cause was twofold: both panels updated local snapshot state optimistically after delete/save, and detail rows/cards used array index keys, which could leave stale DOM content after removing a middle item.
- Updated `LoanPanel` and `InsurancePanel` to reload snapshots from `/api/bank-snapshots` after add/delete operations and to render records with stable content-derived keys instead of index keys.
- Added visible manual delete affordances for both panels: per-record delete buttons are now always visible, and each statement month has a `刪除此月` action.
- Verification: `npm --workspace @hearth/web run check`, `npm --workspace @hearth/api run build`.
